小朋友下樓梯 | NS-SHAFT

大二物件導向程式設計實習：兩人協作開發經典遊戲重製

專案簡介 | Project Overview

  • 開發目的：透過經典遊戲開發實踐 Java 物件導向程式設計 (OOP) 之核心概念。
  • 專案性質：兩人合作開發之課程實習專案。

我的開發角色 | Development Role

  • 遊戲視覺與 UI 實作：負責遊戲封面、GameOver 結算頁面之視覺設計與 UI 元件刻劃。
  • 角色互動與死亡邏輯：實作角色的血量判定機制，包含「掉落陷阱」與「碰撞障礙物扣血」等關鍵死亡條件，確保遊玩公平性。
  • 關卡機制與遊戲模式：設計多元樓梯（左右移動、不規則樓梯），並實作「搜集點數系統」與「無限模式」，增加遊戲的耐玩度。
  • 素材收集與整合：統籌基礎遊戲物件素材及使用Figma做頁面設計，並負責將不同關卡（一至五關）的內容進行最終整合。

技術挑戰 | Technical Challenges

    <
  • 挑戰一：多樣化樓梯的動態行為處理
    💡 解決：針對不同類型的樓梯（如左右移動樓梯、不規則樓梯）實作獨立的行為邏輯，並確保角色在樓梯移動時能順暢地跟隨其位移。
  • 挑戰二：多關卡整合與數據統計
    💡 解決：串聯第一至第五關的遊戲流程，並實作點數搜集與結帳邏輯，確保玩家在進入下一關時，分數能正確累加並顯示在結算畫面中。

專案成果 | Results

  • OOP 應用實踐：成功將物件導向程式設計之理論概念應用於實際遊戲架構，程式碼具備良好的擴展性。
  • 流暢體驗：透過物件池優化，確保高遊戲頻率運作下畫面依然流暢、無卡頓。
  • 團隊協作：導入 Plane 專案管理工具進行任務分配，並使用 Git 進行版本控制，順利完成團隊間的程式碼整合與除錯。
