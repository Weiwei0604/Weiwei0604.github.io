小朋友下樓梯 | NS-SHAFT

大二物件導向程式設計實習：兩人協作開發經典遊戲重製

專案簡介 | Project Overview

  • 開發目的：透過經典遊戲開發實踐OOP核心概念。
  • 專案性質：大二物件導向程式設計實習，兩人合作開發。

我的開發角色 | Development Role

  • 視覺設計與系統整合：使用 Figma 規劃 UI 並實作封面與結算頁。
  • 角色互動邏輯開發：實作血量判定、掉落陷阱與碰撞扣血等核心死亡條件。
  • 關卡機制與遊戲模式：設計多元樓梯行為（如消失、彈跳）、點數系統與無限模式機制。

核心系統說明 | Core Systems

  • 關卡模式：採漸進難度，每關新增一項機制，須集滿點數方可過關。
  • 無限模式：整合全關卡陷阱隨機出現，隨層數增加速度，且累積點數可額外增加一顆心（生命值）
  • 兩者核心差異： 關卡模式重「解鎖與學習新規則」；無限模式重「長時存活與高難度挑戰」。

第壹關：基礎熟悉

實作人物移動、規則樓梯、重力感。

第貳關：威脅初現

實作不規則長短樓梯與尖刺樓梯。

第參關：消失樓梯邏輯

實作左右移動樓梯、消失樓梯（3秒觸發）。

第肆關：反應測試

實作彈跳樓梯、隨機掉落陷阱，考驗玩家反應。

第伍關：極限挑戰

實作隨機速度變化演算法，增加關卡不可預測性。

技術挑戰 | Technical Challenges

  • 樓梯行為：針對移動與消失樓梯編寫獨立邏輯，解決角色位移跟隨的同步問題。
  • 數據串聯：整合五大關卡流程

專案成果 | Results

  • 效能優化：透過物件重複利用機制減少記憶體消耗
  • 開發規範：導入 Plane 管理進度並使用 Git 協作，實現團隊遠端合作。
  • OOP 應用實踐：成功將OOP課程理論概念應用於實際遊戲架構。
